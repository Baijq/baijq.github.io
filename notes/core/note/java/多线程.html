# 线程的周期

线程new出来之后，就是新建状态，start方法调用之后就进入就绪状态,就绪状态的线程拿到cpu资源后开始执行进入运行状态，运行状态的线程wait后进入等待池，或者sleep后睡眠都算阻塞状态，然后条件满足后继续进入就绪状态等待获取cpu资源，拿到cpu进入运行状态，运行结束后进入dead状态等待GC

sleep和wait

  • sleep属于Thread的静态方法，wait是Object的成员方法
  • sleep不会释放锁，wait会释放
  • sleep不需唤醒，休眠到了自动推出阻塞，wait需要用notify、notifyAll唤醒

线程池不建议使用 Executors 去创建，而是通过 ThreadPoolExecutor 的方式

说明：Executors 返回的线程池对象的缺点如下：

1） FixedThreadPool 和 SingleThreadPool： 允许的请求队列长度为 Integer.MAX_VALUE，可能会堆积大量的请求，从而导致 OOM。

2） CachedThreadPool： 允许的创建线程数量为 Integer.MAX_VALUE，可能会创建大量的线程，从而导致 OOM。

使用new ThreadPoolExecutor创建线程池

public void test() {
    //创建线程池
    ThreadPoolExecutor threadPoolExecutor = new ThreadPoolExecutor(
        5,
        10,
        15,
        TimeUnit.SECONDS,
        new ArrayBlockingQueue<>(10),
        Executors.defaultThreadFactory(),
        new ThreadPoolExecutor.AbortPolicy()
    );

    long start = System.currentTimeMillis();
    //异步线程完成任务一
    CompletableFuture<Void> task1 = CompletableFuture.runAsync(() -> {
        String value1 = task1();
    }, threadPoolExecutor);
    //异步线程完成任务二
    CompletableFuture<Void> task2 = CompletableFuture.runAsync(() -> {
        String value2 = task2();
    }, threadPoolExecutor);

    //等待任务1和任务2都完成
    try {
        CompletableFuture.allOf(task1, task2).get();
    } catch (Exception e) {
        e.printStackTrace();
    }
    System.out.println("执行s时间" + (System.currentTimeMillis() - start));
}

原理示意图

线程池饱和拒绝策略-4种

  1. 直接抛出一个异常，默认策略 new ThreadPoolExecutor.AbortPolicy();
  2. 直接丢弃任务 new ThreadPoolExecutor.DiscardPolicy();
  3. 抛弃下一个将要被执行的任务(最旧任务) new ThreadPoolExecutor.DiscardOldestPolicy();
  4. 主线程中执行任务 new ThreadPoolExecutor.CallerRunsPolicy();

常用的阻塞队列

  1. ArrayBlockingQueue 使用数组实现的有界阻塞队列，特性FIFO
  2. LinkedBlockingQueue 使用链表实现的阻塞队列，特性FIFO 可以设置容量，默认为 Integer.MAX_VALUE
  3. PriorityBlockingQueue 使用平衡二叉树实现的具有优先级的无界阻塞队列
